怎知金庸筆下誰最強

之前看過許多人評論金庸武俠小說中最強的高手到底是誰。此問題的答案選項不少，例如獨孤求敗、掃地僧、達摩祖師、葵花寶典作者無名太監、無崖子的師父逍遙子、金輪國師的龍象般若功第十三層、石破天、張無忌、九陰真經創始人黃裳、越女阿青等等，都有人說。我驀地想起國際象棋或圍棋使用的等級分制度。我想若要討論「誰實力最強」這個議題，除了論證外，還應該引入等級分制度會較好，因為等級分是當今公認較公平客觀之評價方法。此外還應考慮所處的人生階段（年輩問題）。有許多絕頂高手是閉關苦練或潛心修行多年，才習得或領悟到絕世武功。但此時也遠離巔峰時期、來日無多了。就算此時要與人爭鋒，恐也心餘力絀。
另一個有趣的觀念是始源、純正的比後代優秀，這種現象除了出現在許多武俠小說、オタク文化劇本外，也包括原教旨主義對宗教經典的推崇、The Shortest History of Europe 提過的古希臘文明。然而我以為現實並非如此。只要知識技術能累積並不斷改進，那後代就會比先祖高明。這也是 What Technology Wants 所揭櫫的主題之一。後輩會衰敗，要不是經歷動亂斷層使文明未能存續，要不就是後人沒能自強不息，致原地踏步甚至不進則退化。

在 Notepad++ 中輸入自訂格式日期時間

因為用的是 Notepad++；過去要輸入日期時都會另外開個記事本，按下 F5 以取得當前時間日期。前幾天想想這樣下去不是辦法，找了一下是否可以依照自訂格式來輸入日期。發現安裝完 Python 與 Python Script，建出一 script； # add_date.py # http://npppythonscript.sourceforge.net/docs/latest/scintilla.html import time if len(editor.getSelText()) > 0: editor.clear() editor.addText(time.strftime("%Y/%m/%d %H:%M:%S")) 設定成 menu item 後設定 shortcut (Plugin commands tab)。若原先已有功能使用此快捷鍵，則將之改掉，就能順暢使用了。這次順便設定自動完成 HTML tag 的結束 tag。

「決不」與「絕不」

發現決不與絕不兩個詞都有人用，像是決不是、絕不是，決不能、絕不能，決不會、絕不會等，這些似乎尚無定論。略看了每日二字－－這樣寫就對了！與「絕不」與「決不」，應寫那一個？、編輯發稿中的高頻易錯字、圖書編輯常見錯字128例幾篇文章；感覺與意志有關，要強調意志堅決的話，該用「決不」。若是要表達沒有這可能性，突出「絕對不可能」的話，則採用「絕不」。雖然可以訂下原則，但之前已經用過成百次，已經無力一個個審查了。此外未來恐怕也不容易每次寫都每次注意到。

凶與兇

最近檢核繁簡轉換，發現一些字詞用法很難判定正俗，現在大概都通用了。「凶」與「兇」為其一。查了一下源流，發現「凶」是不吉利、惡、災禍。例如元兇代表禍亂的源頭，幫兇代表幫助災禍、促進為非作歹。引申為厲害、激烈。「兇」則有恐懼害怕之義（曹人兇懼：曹國人被搞到怕了），假借了「凶」之殘忍粗暴意涵。未來可能會依此為準。現在除了詞典中幾個特別指出的詞彙，如「真兇」、「緝兇」外，基本上都用「凶」。

  • 兇：殘忍粗暴、令人恐懼害怕、恐怖。

VirtualBox 與 Docker 不相容之解決辦法

前兩天又裝了 Docker 想測試，但工作繁忙，雜事多，一直找不到時間。昨天需要先開 VirtualBox 做事，卻發現 Windows 10 虛擬機執行速度異常緩慢，幾乎不動了。Linux Mint 虛擬機雖然也怪怪的，尚沒那麼嚴重。查了一下資源，發現硬碟幾乎不會跑，估計是 CPU （或 RAM）的設定環境在安裝 Docker 期間被改掉了。記得之前上一臺機器也遇過 VirtualBox 與 Docker 不相容的問題，癥結在這兩個根本水火不容。先把 Docker 卸載了再說。遺憾的是問題依舊沒消失。記得上次也遇到相同問題，可惜沒紀錄下怎麼解決的。只好上網找資料了。
依照 Docker + Virtual Box = VT-x is not available 所說，想從 Turn Windows features on or off 關掉 Hyper-V，卻發現本來就沒開。或許是移除 Docker 時，也順便回復了這項設定？依然按照指示執行了 DISM.exe /Online /Disable-Feature:Microsoft-Hyper-V，重開機，沒變好。照虛擬機設定，手動執行 .\VBoxManage.exe modifyvm %VM_NAME% --nested-hw-virt on 嘗試開啟 Enable Nested VT-x/AMD-V 也無效。
又找了一下，才發現還需要從 Turn Windows features on or off 把 Virtual Machine Platform 關起來。記得上次也做過一樣的事，為了預防再次白做工，只好筆記一下。

Microsoft Word Online 語音輸入功能初探

最近持續日記數位化計畫。Google 文件不能直接用語音輸入標點符號，也沒有標點符號快速鍵或者標點符號列。一邊語音輸入一邊繕打標點符號，有時還會漏掉整段句子，對我來說還有些不方便。昨天嘗試了三種語音輸入工具，包括 Microsoft Word Online、Speechnotes 和 VoiceNote。發現 Microsoft Word Online 可以用語音直接輸入標點符號，雖然缺了破折號，刪節號不太標準（是...而非……），要念出上下引號也太繁瑣；但這樣就能低頭一直看著日記，眼睛不用一直在螢幕與日記之間換來換去；可算是比較符合我的需求。我覺得 Microsoft 可利用標點斷句來加強辨識精準度，這點現在無法語音輸入標點符號的 Google 就做不到。比較大的問題是：
  • 辨識程度稍微略遜於 Google 文件，尤其是只輸入單個詞彙的情況（校訂文件時常需要輸入單個詞彙）。輸入的速度也欠奉（令人不敢領教不太滿意），通常是整句話講完過幾秒鐘才會一起出來。Google 文件則是一邊輸入一邊就會跳出來。
  • Google 文件語音辨識模稜兩可時，會出現可選替代字幫助校正。Microsoft 在繁體中文輸入下，現在沒這功能。
  • 語音偵測太過敏感，時不時會自動出現 F、SN 等等奇怪的文字。Google 文件沒這問題。
  • 每次開啟文件、語音輸入，都得要重新設定一次輸入語言，否則會得到簡體中文。就算設定校訂語言，欲防止語音輸入變成簡體字，但兩者似乎沒有關聯。並且設定完只要重新讀取、重開瀏覽器，校訂語言時不時會跳回中文 (中國)。實測似乎全部剪下、清空文件時，就會自動轉回中國設定。也就是說，每次打開文件都得重新設定一次語音輸入的語言。Google 文件設定一次後不會自動跳掉。
  • 輸入中途會自動加入空白。雖然我知道這對以空白鍵為區隔的英文、韓語、阿拉伯語、西班牙語等等頗為方便，但中文可不習慣中間插入空白啊。必須在最終複製前先把所有空白都替換掉。（即便如此，複製時行尾也會多出空白。）但是取代功能必須先尋找再全部取代，因此取代時又多了一個工。其他兩種輸入工具也會自動添加空白字元，Google 文件可在句號前方輸入來迴避這個問題。
  • 不知為何，粗話篩選器（profanity）關不掉，所以在輸入文學作品的名稱、像是白痴，或一般人日常親密對話也會用到的「笨蛋」時就會很麻煩。Google 文件屏蔽的不雅詞彙範圍較小，只會把「幹你娘」之類轉成「幹**」、「班級幹部」變成「班級*部」，一般輸入時幾乎不會遇到。
  • 我改採用 Microsoft Edge 測試，照理來說應該是最配合 Microsoft 365、Office Online 的環境。但剪下、複製、貼上等快速鍵時不時會失常。我的習慣是採用 Ctrl、Shift 配合上 Insert 與 Delete 鍵，不只是 X、C、V 之類。Google 文件在這方面就沒問題。
  • 所設定的語音開關快速鍵與日文輸入法的切換鍵相同，但沒辦法自訂快速鍵，我又沒找到能關掉日文輸入法轉換鍵的方法，所以只能在確定留在中文輸入法時，再使用語音快速鍵。Google 文件沒這問題，但 Google 不像 Microsoft 會在語音辨識停止時跳出聲音，這點 Microsoft 的巧思倒是比較好。就我的使用來說，Speechnotes 會一直聆聽，不會靜音一段時間就自動關閉語音偵測，這是最理想的。Google 不但聆聽的等候時間比Microsoft 短，還有一點不理想的是，常常過段時間不聆聽了，頁面上卻沒什麼變化，圖示仍然停留在聆聽的紅色狀態；說完話才會發現已經失效了。這時想重新輸入語音，必須手動關閉之後再重新開啟聆聽功能，把剛剛那句話重說一次。
有鑑於上面的問題，因此現在的最佳實踐策略，只能 Google 與Microsoft 都開著，首先在 Microsoft Word Online 輸入，之後刪除掉所有空白，複製到 Google 文件來校訂錯字、網頁化（添加 HTML tag）。不過Google 會自動轉換引號「"」為簡體中文標點符號，這點也很頭痛。
